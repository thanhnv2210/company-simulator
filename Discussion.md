# 💬 Discussions

## 1. Why Non-AI First?
- Avoid overengineering
- Build strong domain understanding
- Ensure real value before AI

---

## 2. What is the real product?
Not a simulator, but:
> A system to train engineers to think like decision-makers

---

## 3. Risks
- Becoming static documentation
- Overcomplicating the model
- Lack of user engagement

---

## 4. Open Questions
- How realistic should simulation be?
- How to measure decision quality?
- When to introduce AI?

---

## 5. Future Ideas
- Multiplayer simulation
- Scenario-based challenges
- Integration with real developer workflows

## 6. Why Lightweight Architecture?

- Enables instant usage via GitHub Codespaces
- Reduces setup friction
- Focuses on logic, not infrastructure
- Makes project easy to demo and share

---

## 7. When to Add Backend?

Only when needed:
- Multi-user support
- Persistent state
- AI integration
